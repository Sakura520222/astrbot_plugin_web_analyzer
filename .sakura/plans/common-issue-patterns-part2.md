# 常见问题模式（第二部分）

## 系统解析问题

### 1. 脆弱字符串匹配
**模式**：解析系统命令输出时依赖固定数量空白符匹配

**问题代码**：
```python
# 问题：依赖固定空白符数量
output = subprocess.check_output("id", shell=True).decode()
uid = output.split()[1]  # 假设第二个字段是 UID
```

**解决方案**：
```python
# 正确：按语义提取
output = subprocess.check_output("id", shell=True).decode()
# 使用正则或按语义解析
import re
uid_match = re.search(r'uid=(\d+)', output)
if uid_match:
    uid = uid_match.group(1)
```

### 2. 容器环境假阳性
**模式**：白名单模式可避免误判（如含有 docker 字样的自定义环境变量）

**常见误判**：
- 环境变量包含 "docker" 字样但实际不是 Docker 环境
- `/proc` 解析错误导致误判

**解决方案**：
- 引入白名单机制收敛枚举边界
- 逆向思考"什么情况下会被误判为 Docker"

## 审查流程问题

### 1. 修复遮蔽效应
**模式**：当上一轮遗留问题集中在特定文件时，审查员视线容易被锚定

**后果**：忽略同一文件中其他函数的实际调用变更

**解决方案**：
- 审查时主动检查文件中所有函数的调用变更
- 避免只关注特定代码块的修改

### 2. 增量审查盲区
**模式**：仅看 diff 无法评估新功能在入口文件中的调度逻辑

**后果**：遗漏功能闭环检查

**解决方案**：
- 无历史记忆的增量审查若涉及新功能，须阅读核心入口文件
- 强制检查历史上下文

### 3. 评分通货膨胀
**模式**：连续出现 9/10、10/10 但实际变更微小

**后果**：削弱审查信任度

**解决方案**：
- 严格执行评分校准规则
- 纯版本号/注释/格式化变更评分 ≤ 8/10

### 4. PR描述与实际变更不匹配
**模式**：PR描述中提到的文件未出现在diff中

**后果**：可能遗漏重要变更或存在误导

**解决方案**：
- PR描述中提到的文件必须出现在diff中，否则需追问原因
- 审查时对照PR描述和实际变更文件列表

## 架构设计问题

### 1. 单文件职责过重
**模式**：`analyzer.py` 承载 HTTP 请求、截图、策略选择等多职责

**风险**：变更时牵一发动全身，耦合度高

**改进方向**：
- 考虑职责分离，将不同功能模块化
- HTTP 客户端、浏览器自动化、策略选择等功能独立成模块

### 2. 错误信息丢失
**模式**：异常包装无上下文，损害可观测性

**问题代码**：
```python
# 问题：无上下文的笼统错误
try:
    page.goto(url)
except Exception as e:
    raise Exception("截图生成失败")  # 丢失原始异常信息
```

**解决方案**：
```python
# 正确：传递原始异常信息
try:
    page.goto(url)
except Exception as e:
    raise Exception(f"截图生成失败: {str(e)}") from e
```

## 生命周期管理问题

### 1. 功能不对称
**模式**：插件提供安装功能但无对应的卸载能力

**风险**：用户无法清理残留文件，影响系统整洁

**解决方案**：
- 安装/初始化功能必须检查对应的卸载/清理能力
- 建立 `lifecycle` 标签专门标记安装/卸载/重置类 Issue

### 2. 删除操作风险
**模式**：删除关键目录时未检查前置条件

**常见风险**：
- 进程占用导致删除失败
- 权限不足无法删除
- 删除失败导致插件崩溃

**解决方案**：
```python
# 删除操作前置条件检查
async def uninstall_browser():
    async with asyncio.Lock():
        if _is_installing:
            raise Exception("安装进行中，无法卸载")
        if browser_pool.is_busy():
            raise Exception("截图任务进行中，无法卸载")
        
        try:
            shutil.rmtree(browser_path)
        except PermissionError:
            logger.error("权限不足，无法删除浏览器目录")
            raise Exception("权限不足，请手动删除或以管理员权限运行")
```

### 3. 异步状态同步
**模式**：复合状态检查存在竞态条件

**风险**：`_is_installing` 与浏览器池忙碌状态的复合检查易出错

**解决方案**：
- 使用 `asyncio.Lock` 保护状态检查
- 状态持久化到配置文件
- 插件重启后能恢复状态

## 测试覆盖问题

### 1. 超时逻辑测试缺失
**模式**：涉及超时、重试、降级逻辑的 PR 缺少测试覆盖

**风险**：边界情况未验证，可能导致运行时错误

**解决方案**：
- 要求涉及超时、重试、降级逻辑的 PR 必须附带测试
- 测试应覆盖边界值（0、负数、超大值）

### 2. 配置边界测试缺失
**模式**：配置边界值未在测试中明确处理

**常见遗漏**：
- `timeout=0` 的行为
- `timeout=-1` 的处理
- `None` 值的默认值应用

**解决方案**：
- 配置边界值必须在代码或测试中明确处理
- 建立配置验证测试套件
