# 插件配置

> 来源: https://docs.astrbot.app/dev/star/guides/plugin-config.html

---

AstrBot 支持通过 `_conf_schema.json` 文件为插件定义配置项，用户可以在 WebUI 中方便地修改插件配置。

## 基本用法

在插件根目录下创建 `_conf_schema.json` 文件：

```json
{
  "token": {
    "description": "Bot Token",
    "type": "string"
  },
  "sub_config": {
    "description": "测试嵌套配置",
    "type": "object",
    "hint": "xxxx",
    "items": {
      "name": {
        "description": "testsub",
        "type": "string",
        "hint": "xxxx"
      },
      "id": {
        "description": "testsub",
        "type": "int",
        "hint": "xxxx"
      },
      "time": {
        "description": "testsub",
        "type": "int",
        "hint": "xxxx",
        "default": 123
      }
    }
  }
}
```

## 配置项字段说明

每个配置项支持以下字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| `type` | ✅ | 配置的类型。支持 `string`, `text`, `int`, `float`, `bool`, `object`, `list`, `dict`, `template_list`。当类型为 `text` 时，将会可视化为一个更大的可拖拽宽高的 textarea 组件 |
| `description` | ❌ | 配置的描述。建议一句话描述配置的行为 |
| `hint` | ❌ | 配置的提示信息，表现在右边的问号按钮，鼠标悬浮时显示 |
| `obvious_hint` | ❌ | 配置的 hint 是否醒目显示 |
| `default` | ❌ | 配置的默认值。int 是 0，float 是 0.0，bool 是 False，string 是 ""，object 是 {}，list 是 [] |
| `items` | ❌ | 如果类型是 `object`，需要添加此字段，内容是子 Schema。理论上可以无限嵌套 |
| `invisible` | ❌ | 配置是否隐藏。默认 `false`，设为 `true` 则不在管理面板显示 |
| `options` | ❌ | 下拉列表可选项，如 `"options": ["chat", "agent", "workflow"]` |
| `editor_mode` | ❌ | 是否启用代码编辑器模式（需 AstrBot >= v3.5.10）。默认 false |
| `editor_language` | ❌ | 代码编辑器的代码语言，默认为 `json` |

## 嵌套配置（object 类型）

当 `type` 为 `object` 时，使用 `items` 定义子配置：

```json
{
  "database": {
    "description": "数据库配置",
    "type": "object",
    "items": {
      "host": {
        "description": "主机地址",
        "type": "string",
        "default": "localhost"
      },
      "port": {
        "description": "端口号",
        "type": "int",
        "default": 3306
      }
    }
  }
}
```

## template_list 类型

`template_list` 类型允许用户添加多个相同结构的条目：

```json
{
  "custom_extra_body": {
    "description": "自定义请求体参数",
    "type": "dict",
    "items": {},
    "hint": "用于在请求时添加额外的参数，如 temperature、top_p、max_tokens 等。",
    "template_schema": {
      "temperature": {
        "name": "Temperature",
        "description": "温度参数",
        "hint": "控制输出的随机性，范围通常为 0-2",
        "type": "float",
        "default": 0.6,
        "slider": {"min": 0, "max": 2, "step": 0.1}
      },
      "top_p": {
        "name": "Top-p",
        "description": "Top-p 采样",
        "type": "float",
        "default": 1.0,
        "slider": {"min": 0, "max": 1, "step": 0.01}
      }
    }
  }
}
```

## _special 字段

`_special` 字段仅 v4.0.0 之后可用。常用值包括：

| _special 值 | 返回类型 | 说明 |
|-------------|---------|------|
| `select_provider` | `string` | 选择已配置的模型提供商 |
| `select_provider_tts` | `string` | 选择 TTS 提供商 |
| `select_provider_stt` | `string` | 选择 STT 提供商 |
| `select_persona` | `string` | 选择人设 |
| `select_knowledgebase` | `list` | 选择知识库（支持多选，建议 `type` 设为 `list`，默认值 `[]`） |

以 `select_provider` 为例：

```json
{
  "llm_provider": {
    "description": "选择 LLM 提供商",
    "type": "string",
    "_special": "select_provider"
  }
}
```

## 在插件中读取配置

```python
class Main(Star):
    def __init__(self, context: Context):
        super().__init__(context)
        # 读取配置
        self.token = self.config.get("token", "")

    @filter.command("test")
    async def test(self, event: AstrMessageEvent):
        token = self.config.get("token", "")
        yield event.plain_result(f"Token: {token}")
```

## 配置项国际化（可选）

配置项的 `description`、`hint` 和下拉选项 `labels` 支持按 WebUI 语言显示，详见插件国际化章节。
