# 项目概览 - astrbot_plugin_web_analyzer

## 项目简介
本项目是 AstrBot 的一个插件，主要用于 Web 内容分析，包含网页截图等功能。

## 核心功能
- Web 内容分析
- 网页截图功能
- 多策略截图等待机制
- HTTP 请求与浏览器自动化双客户端管理

## 技术架构

### 技术栈
- **主要语言**：Python
- **核心依赖**：
  - `httpx`：网络请求（超时单位：秒）
  - `playwright`：浏览器自动化（超时单位：毫秒）

### 架构设计
- **分层架构**：配置层 → 入口层 → 消息处理层 → 核心分析层
- **配置驱动**：行为完全由配置决定，灵活性高但测试组合复杂
- **策略模式**：截图等待策略支持 `fixed`、`networkidle`、`smart` 三种模式

## 关键设计模式

### 1. 配置透传模式
- **描述**：超时、重试等参数从配置层 → 构造函数 → 多处异步调用
- **风险**：隐式类型假设（如字符串 `"30"` 参与算术运算）
- **规范**：所有使用 `self.timeout` 进行算术运算的地方，须确保其为 `int/float` 类型

### 2. 双客户端架构
- **描述**：同时管理 HTTP 客户端和浏览器自动化
- **挑战**：两种客户端的超时语义不同（毫秒 vs 秒）
- **规范**：审查时须留意单位转换一致性

### 3. 策略降级机制
- **描述**：截图等待策略支持降级链，确保功能可用性
- **关键点**：`networkidle` 在 SPA 页面可能永远无法触发，必须依赖 `smart` 策略兜底
- **规范**：策略降级必须记录日志，明确包含降级原因和降级目标

### 4. 参数构建收敛
- **描述**：浏览器启动参数统一在构建器中生成
- **规范**：禁止在重试逻辑或初始化逻辑中出现硬编码的散落参数

## 常见问题与注意事项

### 配置相关
- **类型安全**：配置值为字符串时，参与算术运算前必须转换为数值类型
- **边界值处理**：`timeout=0`、负数、`None`、空字符串等边界值必须明确处理
- **配置同步**：配置项变更必须同步更新 `_conf_schema.json`

### 超时与重试
- **硬编码超时**：禁止在浏览器操作中使用硬编码超时值
- **单位一致性**：注意 playwright（毫秒）和 httpx（秒）的单位差异
- **测试覆盖**：涉及超时、重试、降级逻辑的 PR 必须附带测试

### 版本管理
- **双版本号同步**：`main.py` 与 `metadata.yaml` 必须强制同步
- **版本提交规范**：核心修复合入时应立即更新版本号，避免单独的“空版本号提交”
- **依赖版本提升**：必须注明依赖的特定 API/Feature

### 系统解析
- **容器检测**：解析系统命令输出时，严禁依赖固定数量空白符匹配
- **假阳性防范**：引入白名单机制避免误判（如含有 docker 字样的自定义环境变量）
