# 处理消息事件

> 来源: https://docs.astrbot.app/dev/star/guides/listen-message-event.html

---

事件监听器可以收到平台下发的消息内容，可以实现指令、指令组、事件监听等功能。

事件监听器的注册器在 `astrbot.api.event.filter` 下，需要先导入。请务必导入，否则会和 python 的高阶函数 filter 冲突。

```python
from astrbot.api.event import filter, AstrMessageEvent
```

## 消息与事件

AstrBot 接收消息平台下发的消息，并将其封装为 `AstrMessageEvent` 对象，传递给插件进行处理。

### 消息事件

`AstrMessageEvent` 是 AstrBot 的消息事件对象，其中存储了消息发送者、消息内容等信息。

### 消息对象

`AstrBotMessage` 是 AstrBot 的消息对象，其中存储了消息平台下发的消息具体内容，`AstrMessageEvent` 对象中包含一个 `message_obj` 属性用于获取该消息对象。

```python
class AstrBotMessage:
    '''AstrBot 的消息对象'''
    type: MessageType          # 消息类型
    self_id: str               # 机器人的识别id
    session_id: str            # 会话id。取决于 unique_session 的设置。
    message_id: str            # 消息id
    group_id: str = ""         # 群组id，如果为私聊，则为空
    sender: MessageMember      # 发送者
    message: List[BaseMessageComponent]  # 消息链。比如 [Plain("Hello"), At(qq=123456)]
    message_str: str           # 最直观的纯文本消息字符串，将消息链中的 Plain 消息（文本消息）连接起来
    raw_message: object
    timestamp: int             # 消息时间戳
```

其中，`raw_message` 是消息平台适配器的原始消息对象。

### 消息链

`消息链`描述一个消息的结构，是一个有序列表，列表中每一个元素称为`消息段`。

常见的消息段类型有：

- `Plain` — 文本
- `At` — @某人
- `Image` — 图片
- `Record` — 语音
- `Video` — 视频
- `File` — 文件

大多数消息平台都支持上面的消息段类型。

此外，OneBot v11 平台（QQ 个人号等）还支持以下较为常见的消息段类型：

- `Face` — QQ 表情
- `Node` — 合并转发节点
- `Nodes` — 合并转发
- `Poke` — 戳一戳

## 指令注册

使用 `@filter.command("指令名")` 装饰器注册指令：

```python
@filter.command("helloworld")
async def helloworld(self, event: AstrMessageEvent):
    '''这是一个 hello world 指令'''
    yield event.plain_result("Hello!")
```

## 监听所有消息

使用 `@filter.on_message()` 装饰器监听所有消息（不包括自己发送的）：

```python
@filter.on_message()
async def on_message(self, event: AstrMessageEvent):
    message_str = event.message_str
    logger.info(f"收到消息: {message_str}")
```

## 指令参数

指令可以声明参数，AstrBot 会自动从消息中解析：

```python
@filter.command("greet")
async def greet(self, event: AstrMessageEvent, name: str):
    '''向指定的人问好'''
    yield event.plain_result(f"Hello, {name}!")
```

## 过滤消息平台

可以使用 `@filter.platform` 过滤特定平台：

```python
@filter.on_message()
@filter.platform("aiocqhttp")
async def on_qq_message(self, event: AstrMessageEvent):
    # 只处理 QQ 消息
    pass
```

## 指令组

可以将相关的指令组织到一个指令组中：

```python
@filter.command_group("group")
class MyGroup:
    @filter.command("sub")
    async def sub_command(self, event: AstrMessageEvent):
        yield event.plain_result("This is a sub command")
```
